Timestamp is early in its public package life, so focused feedback is especially useful.

Good contributions include:

- Regression tests for parsing and date math edge cases.
- SSR and timezone mismatch examples.
- Documentation improvements where behavior could be clearer.
- Small, well-scoped helpers that keep the package deterministic and framework-agnostic.
